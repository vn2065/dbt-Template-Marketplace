# Distribution Strategy

## Pricing: Free

All templates are free and open source (MIT license).

## Distribution Channels

| Channel | Action |
|---------|--------|
| **GitHub** | Primary distribution — star-able, forkable, discoverable via search |
| **Gumroad** | "Pay what you want" with $0 minimum — captures email for updates |
| **dbt Hub** | Submit packages if templates become installable packages |
| **dbt Slack** | Post in #tools-and-integrations, #analytics-engineering |

## Why Free?

- **Discoverability**: Free tools get shared. Paid tools get pirated or ignored.
- **Community**: Contributors improve the templates over time.
- **Reputation**: Being known as the person who made these builds trust and opens doors.
- **Lead gen**: If you offer consulting or custom work, free templates are the best top-of-funnel.

## Gumroad Setup (Pay What You Want)

- Set minimum price to $0
- Suggested amount: $0 (let buyers choose)
- Enable "pay what you want" — some buyers will tip $5-$20 voluntarily
- Primary value: captures buyer email for future updates/announcements
- Add a "If this saved you time, consider buying me a coffee" note

## GitHub Setup

- Public repository
- Clear README with screenshots of `dbt docs` output
- Topics: `dbt`, `analytics-engineering`, `saas`, `ecommerce`, `fintech`, `open-source`
- `CONTRIBUTING.md` to invite pull requests
- Releases/tags for versioned downloads

## Community Amplification

- Post on r/dataengineering — free tools are welcomed, paid are sometimes flagged
- dbt Slack `#tools-and-integrations` — free tools get more engagement
- LinkedIn/Twitter — frame as "I built this and open-sourced it"
- Data Engineering Weekly newsletter submission
- Awesome-dbt list (GitHub)
